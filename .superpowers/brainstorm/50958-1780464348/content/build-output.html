Build output format

How should the TypeScript/Vue.js project compile to a userscript?

You mentioned the build should overwrite lichess-board-speaker.user.js. Since userscripts are single-file JavaScript with special headers, how should the build process work?

A

Bundled Userscript (Recommended)

Build process bundles TypeScript + Vue components into a single .user.js file

Pros

  • Standard userscript distribution (single file)
  • Works with existing Tampermonkey setup
  • Can use build tools like Vite/Rollup
  • Vue SFC components work normally

Cons

  • Build step required for every change
  • Debugging minified code is harder
B

Dev/Dist Modes

Development uses separate files, production builds to single userscript

Pros

  • Fast iteration in dev mode
  • Source maps for debugging
  • Clean separation of concerns

Cons

  • More complex build configuration
  • Need to test both modes
  • Dev mode may not work with userscript managers

Note: Both approaches produce the same final lichess-board-speaker.user.js file. The difference is whether you have a dev mode or always work with the bundled version.
