Project file structure

How should the TypeScript source code be organized?

Given the decisions so far (Vue for controls, vanilla JS for board overlays, central settings store), how should the source files be organized?

A

By Feature/Domain

Group files by what they do (settings, board, speech, drawing, etc.)

src/
├── main.ts
├── settings/
│ ├── SettingsStore.ts
│ ├── types.ts
│ └── defaults.ts
├── board/
│ ├── BoardReader.ts
│ ├── BoardWatcher.ts
│ ├── Custom3DBoard.ts
│ ├── Dividers.ts
│ └── coordinateUtils.ts
├── speech/
│ ├── SpeechSynthesizer.ts
│ └── textGenerators.ts
├── drawing/
│ ├── DrawingsOverlay.ts
│ └── commandParser.ts
├── flash/
│ ├── FlashOverlay.ts
│ └── flashManager.ts
├── components/ (Vue)
│ ├── ControlPanel.vue
│ ├── ButtonRow.vue
│ ├── PiecesList.vue
│ └── SettingButton.vue
└── __tests__/
├── settings.test.ts
├── boardReader.test.ts
└── speech.test.ts

Pros

  • Related code grouped together
  • Easy to find feature implementation
  • Natural for feature additions

Cons

  • More directories
  • Potentially deeper nesting
B

By Type (Flat)

Group by technical type (components, services, utils)

src/
├── main.ts
├── components/
│ ├── ControlPanel.vue
│ ├── ButtonRow.vue
│ ├── PiecesList.vue
│ └── SettingButton.vue
├── services/
│ ├── SettingsStore.ts
│ ├── BoardReader.ts
│ ├── BoardWatcher.ts
│ ├── Custom3DBoard.ts
│ ├── SpeechSynthesizer.ts
│ ├── DrawingsOverlay.ts
│ ├── FlashOverlay.ts
│ └── Dividers.ts
├── utils/
│ ├── coordinateUtils.ts
│ ├── commandParser.ts
│ └── textGenerators.ts
├── types/
│ └── settings.ts
└── __tests__/
├── settings.test.ts
├── boardReader.test.ts
└── speech.test.ts

Pros

  • Simpler, flatter structure
  • Fewer directories
  • See all services/components at once

Cons

  • services/ becomes large
  • Harder to find related files
  • Less semantic organization

My recommendation: Option A (by feature/domain). With 14+ settings and multiple subsystems, feature-based organization makes it easier to navigate and maintain.
