Testing Strategy & Build Configuration

Unit tests with node:test and bundling to userscript

Testing Approach

What to Test
✓ Settings Store
- update() saves to localStorage and notifies listeners
- get() returns correct values
- load() restores from localStorage
- Multiple listeners all get notified

✓ Board Reading Logic
- pixelsToSquare() converts coordinates correctly
- Handles both white/black perspectives
- Edge cases (0,0), (max,max)

✓ Coordinate Utilities
- squareToPixels(), pixelsToSquare() round-trip
- File/rank conversions (col → 'a'-'h', row → 1-8)

✓ Speech Text Generation
- generateQuadrantText() for all 4 quadrants
- generateAllPiecesText() groups by piece type
- generateColorText() filters by color

✓ Drawing Command Parser
- parseDrawCommand('-e4') → circle
- parseDrawCommand('-e2e4') → arrow
- parseDrawCommand('-a1,b2c3,d4') → multiple
- Invalid input handling

✗ Skip (manual testing only)
- Vue component rendering
- Three.js canvas output
- SVG rendering
- DOM manipulation
- MutationObserver behavior
- Web Speech API

Example test with node:test:

// settings/SettingsStore.test.ts
import { describe, it, beforeEach } from 'node:test'
import assert from 'node:assert'
import { SettingsStore } from '../settings/SettingsStore'

describe('SettingsStore', () => {
  let store: SettingsStore

  beforeEach(() => {
    store = new SettingsStore()
    // Mock localStorage
    global.localStorage = {
      getItem: () => null,
      setItem: () => {},
    }
  })

  it('updates setting and notifies listeners', () => {
    let notified = false
    store.onChange(() => { notified = true })

    store.update('blur', 3)

    assert.strictEqual(store.get('blur'), 3)
    assert.strictEqual(notified, true)
  })

  it('notifies all listeners', () => {
    let count = 0
    store.onChange(() => { count++ })
    store.onChange(() => { count++ })

    store.update('blur', 5)

    assert.strictEqual(count, 2)
  })
})

Run tests:
npm test
# or
node --test src/__tests__/**/*.test.ts

Build Configuration

Vite Configuration
// vite.config.ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  build: {
    lib: {
      entry: './src/main.ts',
      name: 'LichessBoardSpeaker',
      formats: ['iife'], // Single file bundle
      fileName: () => 'lichess-board-speaker.user.js'
    },
    outDir: '.', // Output to project root
    minify: false, // Readable code as requested
    sourcemap: 'inline', // For debugging
    rollupOptions: {
      external: [], // Bundle everything
      output: {
        banner: generateUserscriptHeader(),
        format: 'iife',
        inlineDynamicImports: true
      }
    }
  }
})

function generateUserscriptHeader() {
  const pkg = require('./package.json')
  return `// ==UserScript==
// @name lichess-board-speaker
// @description Blindfold chess training tool for lichess.org
// @version ${pkg.version}
// @match *://lichess.org/*
// @require https://cdnjs.cloudflare.com/ajax/libs/three.js/r128/three.min.js
// @grant none
// @inject-into content
// @updateURL https://cdn.jsdelivr.net/gh/dylan-chong/lichess-utils@main/lichess-board-speaker.user.js
// @downloadURL https://cdn.jsdelivr.net/gh/dylan-chong/lichess-utils@main/lichess-board-speaker.user.js
// ==/UserScript==`
}

Development Workflow

1. Make changes to TypeScript/Vue files
   src/board/Custom3DBoard.ts
   src/components/ControlPanel.vue

2. Run build in watch mode
   npm run dev
   → Watches for changes
   → Rebuilds lichess-board-speaker.user.js on save

3. Reload lichess.org page
   → Tampermonkey detects updated .user.js
   → Reloads script automatically
   → Test changes immediately

For testing:
   npm test # Run once
   npm run test:watch # Watch mode

Before commit:
   npm run type-check # Verify TypeScript types
   npm test # Run all tests
   npm run build # Production build

Key points:

  • Vite bundles everything (TypeScript + Vue + Three.js) into single .user.js file
  • No minification (readable code with inline source maps)
  • Userscript header auto-generated with version from package.json
  • Tests run with native node:test (no Jest/Vitest needed)
  • tsx loader handles TypeScript in tests
