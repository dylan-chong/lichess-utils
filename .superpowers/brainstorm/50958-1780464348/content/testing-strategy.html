Testing strategy with node:test

What should unit tests cover in this architecture?

You specified node:test for unit testing. Given the architecture (Vue controls + vanilla JS board overlays), what should be the testing focus?

A

Core Logic & State (Recommended)

Test settings store, board reading logic, and business logic. Skip DOM/rendering.

Test Coverage:
✓ SettingsStore (get/set/save/load)
✓ BoardReader (parse piece positions)
✓ Coordinate conversion logic
✓ Speech text generation
✓ Drawing command parser
✓ Settings validation

Skip:
✗ Vue component rendering
✗ Three.js canvas output
✗ SVG overlay rendering
✗ MutationObserver DOM watching

Pros

  • Fast tests (no DOM, no browser)
  • Focus on business logic bugs
  • Easy to write and maintain
  • Tests survive refactoring

Cons

  • No coverage of rendering bugs
  • Won't catch DOM integration issues
B

Full Coverage with JSDOM

Test everything including Vue components and DOM interactions with JSDOM

Test Coverage:
✓ SettingsStore
✓ BoardReader
✓ Business logic
✓ Vue component rendering
✓ Button click handlers
✓ DOM manipulation
✓ MutationObserver behavior

Pros

  • Comprehensive coverage
  • Catches more integration bugs
  • Tests closer to real usage

Cons

  • Slower tests
  • More setup complexity (JSDOM, Vue test utils)
  • Tests more brittle (break on UI changes)
  • Harder to debug

My recommendation: Option A. Testing core logic gives you confidence in correctness without the complexity of DOM testing. You'll manually test the UI in the browser anyway since this is a userscript.
