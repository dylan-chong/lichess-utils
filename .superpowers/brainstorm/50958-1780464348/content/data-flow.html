Data Flow & State Management

How data moves through the system

Settings Flow

1. User clicks "Blur: 0px" button
  ↓
2. SettingButton.vue emits click
  currentValue = 0px
  options = [0px, 1px, 2px, 3px, 4px, 6px, 8px]
  nextValue = 1px
  settingsStore.update('blur', 1)
  ↓
3. SettingsStore.update('blur', 1)
  this.blur = 1
  this.save() // localStorage.setItem('lichess-board-speaker-settings', JSON.stringify(this))
  this.emit('settings:changed', this.getAll())
  ↓
4. All managers receive event
  boardManager.reapply(settings) → updates blur on 3D canvas
  speechManager.reapply(settings) → (no change needed)
  flashManager.reapply(settings) → (no change needed)
  ↓
5. Vue components re-render
  SettingButton shows "Blur: 1px"

Board Change Flow

1. User makes a move on lichess board
  ↓
2. MutationObserver detects DOM change
  BoardWatcher sees piece.style.transform changed
  boardWatcher.emit('board:changed')
  ↓
3. Multiple systems react
  FlashManager.onBoardChange()
    → overlay.hide() // Show board
    → setTimeout(() => overlay.show(), flashDuration)
    → Reset interval timer
  ↓
  PiecesList.vue (watching board:changed)
    → pieces = boardReader.readPiecePositions()
    → Re-render list
  ↓
  Custom3DBoard (if enabled)
    → pieces = boardReader.readPiecePositions()
    → render(pieces) // Update 3D scene

Keyboard Command Flow

1. User types "pwk" in move input
  ↓
2. Input handler detects command
  const input = document.querySelector('.keyboard-move input')
  input.addEventListener('input', (e) => {
    if (e.target.value.startsWith('p')) {
      handleButtonCommand(e.target.value)
    } else if (e.target.value.startsWith('-')) {
      handleDrawCommand(e.target.value)
    }
  })
  ↓
3. Speech command handler
  handleButtonCommand('pwk')
    → speechManager.speakQuadrant('white-kingside')
    → pieces = boardReader.readPiecePositions()
    → text = generateQuadrantText(pieces, 'wk')
    → synthesizer.speak(text, settingsStore.get('speakRate'))
    → input.value = '' // Clear input

Drawing command example: "-e4"
  handleDrawCommand('-e4')
    → commands = parseDrawCommand('-e4') // [{circle: 'e4'}]
    → drawingManager.draw(commands)
    → input.value = ''

Page Navigation Flow

User navigates from puzzle → game
  ↓
Detect page change (URL change or full board DOM replacement)
  ↓
Tear down current instance
  boardManager.destroy()
    → boardWatcher.stop()
    → board3D.destroy() // Remove canvas
    → dividers.destroy() // Remove SVG
  flashManager.destroy() // Clear timers
  drawingManager.destroy() // Remove SVG
  app.unmount() // Vue cleanup
  ↓
Wait for new page DOM
  waitForElement('.keyboard-move')
  ↓
Reinitialize (same as startup)
  Settings persist (already in localStorage)
  Create new managers, mount Vue, apply settings

State Management Details

SettingsStore Implementation
interface Settings {
  speakRate: number // 0.2, 0.5, 0.7, 1.0, 1.1, 1.2
  piecesListEnabled: boolean
  dividersEnabled: boolean
  customBoardEnabled: boolean
  obfuscationsEnabled: boolean
  parallax: number // 0, 20, 30, 40, 50, 60, 65, 70, 80
  hoverMode: string // 'off', 'small', 'large', 'super'
  pieceStyle: string // 'icons', '3d', 'checker', 'checker-grey', 'blindfold'
  blur: number // 0, 1, 2, 3, 4, 6, 8
  blackSegments: string // 'none', '1/4', '1/2', '3/4', '4/4'
  blackSegmentsTiming: string // 'rotate-10s', 'rotate-30s', 'rotate-60s', 'dont-rotate'
  flashModeEnabled: boolean
  flashDuration: number // 0.3, 0.5, 1, 2 (seconds)
  flashInterval: number // 1, 3, 5, 10, 30, 60 (seconds)
}

const STORAGE_KEY = 'lichess-board-speaker-settings'

class SettingsStore extends EventEmitter {
  private settings: Settings

  constructor() {
    super()
    this.settings = getDefaultSettings()
  }

  load() {
    const stored = localStorage.getItem(STORAGE_KEY)
    if (stored) {
      this.settings = { ...this.settings, ...JSON.parse(stored) }
    }
  }

  save() {
    localStorage.setItem(STORAGE_KEY, JSON.stringify(this.settings))
  }

  update(key: keyof Settings, value: any) {
    this.settings[key] = value
    this.save()
    this.emit('settings:changed', this.getAll())
  }

  get(key: keyof Settings) {
    return this.settings[key]
  }

  getAll(): Settings {
    return { ...this.settings }
  }
}

Key insight: The settings store is completely decoupled from managers. Managers don't modify settings, they only read and react. This makes the data flow unidirectional and predictable.
