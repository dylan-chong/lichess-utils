Rust-Based Tooling Configuration

SWC for compilation, Biome for linting/formatting

Rust Tools We'll Use

1. SWC - Rust TypeScript/JSX Compiler
Replaces: esbuild's TypeScript transform
Purpose: Compiles TypeScript + JSX → JavaScript
Speed: 20-70x faster than Babel, slightly faster than esbuild
Integration: Via @swc/core + unplugin-swc

What it does:
• Strips TypeScript types
• Transforms JSX to Preact's h() calls
• Minification (optional, we're skipping)
• Faster HMR (Hot Module Replacement) in dev mode
2. Biome - Rust Linter + Formatter
Replaces: ESLint + Prettier
Purpose: Code quality checks + consistent formatting
Speed: 100x faster than ESLint, 35x faster than Prettier
Single binary: No node_modules bloat

What it does:
• Linting (catches bugs, enforces patterns)
• Formatting (consistent code style)
• Import sorting
• Single config file (biome.json)
• Editor integration (VS Code, etc.)

import { defineConfig } from 'vite'
import swc from 'unplugin-swc'

export default defineConfig({
  plugins: [
    swc.vite({
      jsc: {
        parser: {
          syntax: 'typescript',

          react: {
            pragma: 'h',
            pragmaFrag: 'Fragment',
            importSource: 'preact',

Development Workflow with Rust Tools

Typical Development Session
1. Start dev build (watch mode):
npm run dev
→ SWC compiles TypeScript/JSX super fast
→ Vite rebuilds on file changes (instant HMR)
→ Output: lichess-board-speaker.user.js

2. Run tests in watch mode (parallel terminal):
npm run test:watch
→ SWC compiles test files
→ Vitest re-runs affected tests instantly

3. Before committing:
npm run lint:fix # Biome fixes issues automatically
npm run format # Biome formats code
npm run type-check # TypeScript type checking
npm test # Run all tests once

4. Build for production:
npm run build
→ SWC compiles everything
→ Vite bundles to single .user.js file
→ No minification (readable output)
→ Inline source maps for debugging

Speed Comparison (Estimated)

Task esbuild SWC Improvement
Cold build 1.2s 0.8s 33% faster
Incremental rebuild 300ms 150ms 50% faster
Test compilation 800ms 400ms 50% faster
Linting (ESLint vs Biome) 2.5s 25ms 100x faster
Formatting (Prettier vs Biome) 800ms 20ms 40x faster

Times are estimates for this project size (~40 files). Your mileage may vary.

Editor Integration

VS Code Setup
Install Biome extension:
• "Biome" by biomejs

.vscode/settings.json:
{
  "editor.defaultFormatter": "biomejs.biome",
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "quickfix.biome": "explicit",
    "source.organizeImports.biome": "explicit"
  }
}

Result: Auto-format and auto-fix on save (instant with Rust)

Summary:

  • SWC - 33-50% faster compilation than esbuild
  • Biome - 40-100x faster linting/formatting than ESLint/Prettier
  • Single binary - Biome has no node_modules overhead
  • Better DX - Instant feedback in editor and terminal
  • Same output - Still produces single .user.js file

You'll notice: Faster rebuilds when editing files, instant lint/format feedback, snappier tests.
