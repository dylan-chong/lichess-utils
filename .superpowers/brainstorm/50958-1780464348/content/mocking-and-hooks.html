Mocking with Simone & Pre-commit Hooks

Type-safe mocking and automated quality checks

Simone - Type-Safe Mocking

Why Simone over vi.mock()
Simone advantages:
• Full TypeScript inference (args + return types checked)
• Sinon-style expectations (explicit, declarative)
• Strict by default (unexpected calls throw immediately)
• Sequential expectations (call order verified)
• Cleaner API than vi.mock() for function modules

vs Vitest vi.mock():
• vi.mock() - loose, manual setup, easy to break types
• Simone - strict, auto-typed, catches mistakes at compile + runtime

Testing Patterns with Simone

handlers/ - Mocking Dependencies
import { mockModule } from 'simone'
import { handleSpeechCommand } from './handleSpeechCommand'

const boardReader = mockModule<typeof import('../dom/boardReader')>('../dom/boardReader')
const synthesizer = mockModule<typeof import('../browser/speechSynthesizer')>('../browser/speechSynthesizer')

it('speaks white kingside quadrant', () => {
  // Set up expectations in order
  boardReader.expects('readPiecePositions').returns([
    { color: 'white', type: 'pawn', square: 'e2' },
    { color: 'white', type: 'knight', square: 'g1' }
  ])

  synthesizer.expects('speak')
    .withArgs('e2 white pawn. g1 white knight.', 0.5)
    .returns(undefined)

  // Call handler
  handleSpeechCommand('wk')

  // Simone automatically verifies all expectations consumed
})
effects/ - Mocking Handlers
import { mockModule } from 'simone'
import { settings } from '../settings/settingsStore'
import { init, destroy } from './on3dBoard'

const update3dBoard = mockModule<typeof import('../handlers/update3dBoard')>('../handlers/update3dBoard')

it('calls handler when customBoardEnabled changes', () => {
  init()

  // Expect handler called with current settings
  update3dBoard.expects('default')
    .withArgs(expect.objectContaining({
      customBoardEnabled: true,
      parallax: 0
    }))
    .returns(undefined)

  // Trigger change
  settings.customBoardEnabled.value = true

  destroy()
})
browser/ - Mocking Web APIs
// For Web APIs, still use vi.stubGlobal (Simone is for modules)
import { vi } from 'vitest'

it('calls speechSynthesis API', () => {
  const mockSpeak = vi.fn()
  vi.stubGlobal('speechSynthesis', {
    speak: mockSpeak,
    cancel: vi.fn()
  })

  synthesizer.speak('test', 1.0)

  expect(mockSpeak).toHaveBeenCalled()
})

Pre-commit Hooks

Why Pre-commit Hooks
Prevent broken commits:
• Compilation errors
• Failing tests
• Type errors
• Linting issues
• Unformatted code

Tool: Husky
• Standard Git hooks manager
• Runs scripts before git commit
• Fast (only checks staged files with lint-staged)

Pre-commit Flow

When you run: git commit

1. lint-staged runs (on staged files only)
   → Biome checks and fixes issues
   → Biome formats code
   → ~50ms (fast!)

2. Type checking
   → tsc --noEmit
   → ~1-3s

3. Tests
   → vitest run
   → ~1-2s (with SWC)

4. Build
   → vite build
   → ~0.8s (with SWC)

Total: ~3-7s

If any step fails → commit blocked
If all pass → commit proceeds

Bypassing Hooks (Emergency)

If you absolutely need to commit without checks:
git commit --no-verify -m "emergency fix"

Use sparingly! Pre-commit hooks catch bugs before they enter history.
