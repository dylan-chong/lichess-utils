Vue.js integration with lichess DOM

How should Vue components interact with the lichess.org page?

The userscript runs on lichess.org and needs to:

  • Insert UI controls into lichess's existing DOM (.keyboard-move container)
  • Overlay the chessboard (cg-board, cg-container)
  • Read piece positions from lichess's board DOM
  • Detect board changes with MutationObserver

How should Vue fit into this?

A

Vue for Controls Only (Recommended)

Vue renders the button UI, vanilla JS handles board interactions

// Vue: Button controls & pieces list
<ControlPanel />
<ButtonRow />
<PiecesList />

// Vanilla JS: Board overlays
- Custom3DBoard (Three.js canvas)
- DrawingsOverlay (SVG annotations)
- FlashOverlay (black screen)
- Dividers (SVG lines)
- BoardReader (read lichess DOM)
- MutationObserver (detect changes)

Pros

  • Vue excels at button UI and reactive state
  • Board overlays don't need Vue overhead
  • Clear separation: UI vs canvas/SVG rendering
  • Easier to port existing Three.js code

Cons

  • Two paradigms (Vue + vanilla JS)
  • Need to coordinate between them
B

Vue Everywhere

Use Vue for all UI including board overlays and canvas management

// Everything in Vue:
<App>
<ControlPanel />
<BoardOverlays>
<Custom3DBoard />
<DrawingsOverlay />
<FlashOverlay />
<Dividers />
</BoardOverlays>
</App>

Pros

  • Single paradigm throughout
  • Consistent component lifecycle
  • Vue reactivity everywhere

Cons

  • Vue overhead for canvas/SVG rendering
  • More complex component tree
  • Three.js integration trickier
  • Harder to reason about performance

My recommendation: Option A. Vue is great for button controls and reactive lists, but board overlays (canvas, SVG) are better handled with vanilla JS/Three.js.
