Redesigned Architecture with Preact

Lightweight (~3kb), reactive, with signals for state management

Why Preact Changes the Design

Preact Signals provide fine-grained reactivity without manual callbacks:

  • Settings become signals - components auto-update when they change
  • No manual listener management (no onChange() callbacks)
  • Simpler than the callback pattern we had before
  • Tiny runtime (~3kb for Preact + ~2kb for signals)

Hooks simplify side effects and lifecycle:

  • useEffect() for reacting to setting changes in managers
  • useComputed() for derived state (e.g., which buttons to show)
  • No component lifecycle methods needed

High-Level Architecture (Revised)

System Components
1. Entry Point Layer (main.ts)
- Initializes application
- Creates signal-based settings store
- Sets up managers with effect() to watch settings
- Mounts Preact root component

2. State Management Layer (settings/)
- Settings as Preact signals (reactive primitives)
- localStorage persistence
- No manual event emitters needed

3. UI Layer (components/ - Preact TSX)
- Functional components with hooks
- Auto-react to signal changes
- JSX for declarative rendering

4. Board Interaction Layer (feature modules)
- Same as before: board/, speech/, drawing/, flash/
- Use effect() to watch settings signals
- Reapply when signals change

Preact Signals Pattern

How Signals Work
import { signal, effect, computed } from '@preact/signals-core'

// Create a reactive signal
const count = signal(0)

// Read value
console.log(count.value) // 0

// Update value (triggers reactions)
count.value = 5

// React to changes
effect(() => {
  console.log('Count changed:', count.value)
})

// Derived state
const doubled = computed(() => count.value * 2)

In Preact components:
function Counter() {
  // Component auto-re-renders when count changes
  return <div>Count: {count}</div>
}

Key difference from callback pattern:
- No manual listener registration
- No notifyListeners() calls
- Signals automatically track dependencies
- Less boilerplate code

Result: Simpler than the callback pattern, more reactive than manual updates, tiny bundle size (~5kb total for Preact + signals).
